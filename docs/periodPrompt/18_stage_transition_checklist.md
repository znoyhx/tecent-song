# 阶段切换检查表

每个阶段完成后，必须使用本检查表判断是否允许进入下一阶段。

## 通用检查

| 检查项 | 通过标准 | 结果 |
| --- | --- | --- |
| 自测是否完成 | 阶段 Prompt 要求的测试均已执行，或明确说明无法执行原因 | 待填 |
| 人工验收是否完成 | 用户可按报告步骤复现效果 | 待填 |
| 是否仍然全中文 | 用户可见文本无英文残留 | 待填 |
| 是否破坏 Mock Demo | Mock 模式仍可启动并通关或主流程可用 | 待填 |
| 是否破坏真实 AI | 已接入阶段后真实 AI 仍可测试或 fallback | 待填 |
| 是否保留 fallback | AI/图片/API 失败时有中文兜底 | 待填 |
| 是否泄露 API Key | 前端、日志、报告、Markdown 无完整 Key | 待填 |
| 是否需要强模型审查 | 高风险阶段必须审查 | 待填 |
| 是否允许进入下一阶段 | 所有阻塞项通过 | 待填 |

## 阶段 0 -> 阶段 1

必须确认：

- 已输出当前仓库状态。
- 已确认前端/后端是否存在。
- 已确认 Key 文件或环境变量存在性但未泄露。
- 已确认缺失启动命令和测试情况。

## 阶段 1 -> 阶段 2

必须确认：

- Mock Demo 可从开始到结局。
- 至少 5 场景、4 NPC、15 线索、5 结局数据存在。
- 线索栏和红色线索可用。
- UI 全中文。

## 阶段 2 -> 阶段 3

必须确认：

- 前端布局完整。
- API client 工作。
- Loading/Error 全中文。
- 前端没有业务越权推进阶段。

## 阶段 3 -> 阶段 4

必须确认：

- 后端 API 合约稳定。
- Mock 对话、调查、选择、结局可用。
- 后端测试通过。
- 错误响应中文。

## 阶段 4 -> 阶段 5

必须确认：

- DeepSeek 真实调用成功至少一次。
- AI JSON 可解析。
- Mock fallback 保留。
- Key 未泄露。
- 需要强模型审查。

## 阶段 5 -> 阶段 6

必须确认：

- 状态机由后端控制。
- 15 线索路径可测试。
- 关键线索影响阶段或结局。
- 出示线索改变 NPC 回复。
- 需要强模型审查。

## 阶段 6 -> 阶段 7

必须确认：

- 监管器拦截现代词、错朝代、剧透、越权、非法线索、非 JSON、阶段跳跃、英文文本。
- repair/fallback 可用。
- 需要强模型审查。

## 阶段 7 -> 阶段 8

必须确认：

- RAG 关键词检索可用。
- 明代知识不会召回错朝代内容。
- RAG 失败不阻塞对话。
- 需要强模型审查。

## 阶段 8 -> 阶段 9

必须确认：

- 硅基流动图片生成真实测试有记录，或失败有明确 fallback。
- 图片保存路径清晰。
- 前端可展示图片或占位图。
- Key 未泄露。

## 阶段 9 -> 阶段 10

必须确认：

- 5 个结局可触发。
- 结局由规则决定，不由 AI 随机决定。
- 历史回声中文。
- 至少 3 条结局路径测试通过。
- 需要强模型审查。

## 阶段 10 完成判定

项目可路演必须满足：

- Mock 模式可完整演示。
- 真实 AI 模式可展示至少一次 NPC 对话。
- 图片生成或占位图演示可用。
- 监管器演示可用。
- 全中文 UI。
- 无 Key 泄露。
- 有 3 分钟路演脚本。
