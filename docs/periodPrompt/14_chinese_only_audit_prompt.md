# 中文化专项审计 Prompt

## 1. 你的角色

你是中文化专项审计模型。你的任务是扫描并修复所有游戏内用户可见英文，确保《史隙》玩家界面全中文。

## 2. 必须先阅读的文档

- `CODEBUDDY.md`
- `docs/periodPrompt/00_global_rules.md`
- `docs/develop/17_testing_and_acceptance.md`
- 当前前端代码
- 当前 Mock 数据
- 当前 fallback 文案
- 当前结局、线索、场景数据

## 3. 阶段目标

必须完成：

- 扫描前端用户可见文本。
- 扫描 Mock 数据。
- 扫描 AI fallback 文案。
- 扫描错误提示。
- 扫描结局和线索。
- 扫描图片 prompt 中可能展示给用户的文本。
- 输出英文残留列表。
- 修复英文残留。
- 修复后再次检查。

## 4. 当前阶段禁止事项

- 禁止修改业务逻辑。
- 禁止改 API 字段名。
- 禁止修改变量名来追求中文。
- 禁止把代码关键字误判为用户文案。
- 禁止跳过二次检查。

## 5. 允许修改的文件范围

- 前端组件中的用户可见文案。
- Mock 数据中的展示文本。
- fallback 文案。
- 错误提示文案。
- 结局、线索、场景中文数据。
- `docs/periodPrompt/reports/chinese_only_audit_report.md`

## 6. 不允许修改的文件范围

- API 字段名。
- 文件名。
- 变量名。
- 密钥文件。
- 状态机规则。

## 7. 具体开发任务

### 任务 1：扫描英文残留

- 任务目标：找出用户可见英文。
- 搜索对象：前端 JSX、Mock JSON、错误提示、fallback、结局、线索。
- 输出：英文残留清单。
- 验收标准：每条列出文件和上下文。

### 任务 2：判断是否用户可见

- 任务目标：区分代码字段和 UI 文案。
- 输出：需要修复 / 不需修复列表。
- 验收标准：不错误修改 API 字段。

### 任务 3：修复中文文案

- 任务目标：将用户可见英文替换为自然中文。
- 输出：修改文件。
- 验收标准：UI 含义不变。

### 任务 4：二次检查

- 任务目标：修复后再次扫描。
- 输出：二次检查结果。
- 验收标准：无用户可见英文残留。

## 8. 数据与接口要求

不得修改接口字段，例如 `session_id`、`clue_id`、`npc_id` 可以保留英文。只修复展示文本。

## 9. 中文化要求

重点禁止用户可见：`Start`、`Loading`、`Error`、`Submit`、`Continue`、`Inventory`、`Clue`、`NPC`、`Session`、`Choice`、`Ending`、`Back`、`Next`。

## 10. AI 接入要求

检查 AI fallback 和 prompt 中要求“输出给用户的内容必须中文”。不需要真实调用 AI，除非用户要求验证。

## 11. 模型必须执行的自测步骤

- 搜索常见英文 UI 词。
- 搜索 JSX 字符串。
- 搜索 JSON 展示字段。
- 运行构建或类型检查。
- 启动页面人工查看主流程。
- 确认按钮、错误、Loading 全中文。

## 12. 人工验收方式

1. 启动前端。
2. 打开开始页。
3. 查看所有按钮和标题。
4. 进入游戏页，查看状态栏、线索栏、对话框。
5. 触发 Loading 和 Error。
6. 进入结局页。
7. 确认玩家看得到的内容全中文。
