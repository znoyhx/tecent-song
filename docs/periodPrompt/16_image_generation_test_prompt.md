# 图片生成测试 Prompt

## 1. 你的角色

你是硅基流动图片生成测试模型。你的任务是验证图片生成 API 可真实调用、图片可保存、前端可展示，失败时 fallback 生效。

## 2. 必须先阅读的文档

- `CODEBUDDY.md`
- `docs/help/Imagegenerate.md`
- `docs/develop/16_visual_asset_plan.md`
- `docs/develop/17_testing_and_acceptance.md`
- `docs/periodPrompt/00_global_rules.md`
- 当前图片生成客户端和资源展示代码

## 3. 阶段目标

必须完成测试：

- 检查硅基流动 Key 是否存在但不打印。
- 调用真实图片生成接口。
- 生成明代书坊背景。
- 生成 NPC 图。
- 生成线索物品图。
- 保存图片。
- 前端展示。
- 失败 fallback。
- 输出测试记录和人工查看方法。

## 4. 当前阶段禁止事项

- 禁止前端直接调用图片生成 API。
- 禁止打印完整 Key。
- 禁止将 Key 写入报告。
- 禁止只用占位图冒充真实生成。
- 禁止生成现代或错朝代视觉。
- 禁止图片失败导致游戏崩溃。

- `docs/develop/*.md`。

## 7. 具体开发任务

### 任务 1：Key 存在性检查

- 任务目标：确认图片生成 Key 可用。
- 输出：存在/不存在。
- 验收标准：不输出 Key。

### 任务 2：真实生成三类图片

- 任务目标：生成场景、NPC、线索图。
- 输入：中文历史悬疑 prompt。
- 输出：本地图片文件。
- 验收标准：至少一次真实 API 返回成功；失败要记录原因。

### 任务 3：保存和展示

- 任务目标：下载图片并放入规范路径。
- 输出：资源路径。
- 验收标准：前端可显示。

### 任务 4：fallback 测试

- 任务目标：图片缺失或 API 失败时显示占位图。
- 验收标准：页面不崩溃，中文提示可见。

## 8. 数据与接口要求

测试 prompt 示例：

```text
明代雨夜书坊后院，旧书箱烧毁，灰烬与积水，远处侦缉人员剪影，低饱和国风，暗色历史悬疑，视觉小说背景，无现代元素
```

## 9. 中文化要求

前端展示图片标题、生成失败提示、占位说明必须中文。Prompt 可包含模型风格词，但用户可见说明必须中文。

## 10. AI 接入要求

必须真实调用硅基流动图片生成 API。若 Key 不存在或接口不可达，必须停止并反馈阻塞，不得用占位图声称真实生成成功。

## 11. 模型必须执行的自测步骤

- 检查 Key 存在性但不打印。
- 调用图片生成接口。
- 保存图片到本地。
- 验证文件存在且非空。
- 启动前端查看展示。
- 模拟缺图 fallback。
- 检查报告无 Key。

## 12. 人工验收方式

1. 打开测试报告。
2. 查看三类图片保存路径。
3. 在文件系统打开图片。
4. 启动前端进入明代书坊场景。
5. 确认背景、NPC 或线索图展示。
6. 临时移走一张图，刷新页面，确认占位图显示。
