## 3. 当前主流程演示链路

- 前端入口：`App` 启动后调用 `GET /api/health` 与 `GET /api/dynasties`；选择“明代”后进入 `ScenarioGenerationPage`；点击进入事件后调用 `POST /api/session/start` 并渲染 `GamePage`。
- 后端接口顺序：`GET /api/health` → `GET /api/dynasties` → `GET /api/roles` → `POST /api/session/start` → `POST /api/investigate` / `POST /api/dialogue` → `GET /api/session/{session_id}` → `POST /api/choice` → `POST /api/ending/resolve`。
- 主路线：明代 / 书坊学徒 / 书坊焚稿案，从书坊前厅进入后院火场，获得烧焦残页与红印纸角，盘问阿沈与顾闻，推进到关键抉择，选择“暗助顾闻护出残页”，进入“暗藏残页”。
- 结局路线：已通过测试覆盖自保、秩序、真相、悲剧，并保留隐藏结局确定性测试。
- 回退路线：后端不可用时首页显示中文离线提示；模型不可用或输出不合规时使用本地中文回退；图片缺失时使用视觉占位或剪影；历史回声失败时使用本地中文模板。

## 11. 人工验收方法

1. 启动后端：

```powershell
cd backend
python -m uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

2. 启动前端：

```powershell
cd frontend
npm run dev -- --host 127.0.0.1 --port 5173
```

3. 打开 `http://127.0.0.1:5173/`。
4. 选择“明代”，生成本局事件并进入书坊焚稿案。
5. 打开右侧“地点 / 调查”，按路演路线获得烧焦残页、红印纸角，盘问阿沈和顾闻。
6. 进入关键抉择后分别选择三条验收路线：将残页交给陆峥、暗助顾闻护出残页、焚尽余稿。
7. 每次结局页应展示中文标题、摘要、正文、历史回声、众人去向与重新开始按钮。
8. 查看左上“演示证据”面板：确认只显示安全摘要、监管状态、本地回退状态与历史回声来源。
9. 判断智能模型是否真实参与：以演示证据面板和结局页来源文案为准；若显示本地回退，则如实说明回退，不冒充真实成功。
