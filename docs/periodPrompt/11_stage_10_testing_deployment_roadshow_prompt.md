# 阶段 10：测试、部署与路演 Prompt

## 1. 你的角色

你是最终联调、测试、部署和路演准备执行模型。你的任务是验证项目可运行、可通关、可真实 AI 演示、可图片生成演示，并输出最终验收报告。

## 2. 必须先阅读的文档

- `CODEBUDDY.md`
- `docs/develop/17_testing_and_acceptance.md`
- `docs/develop/20_development_roadmap.md`
- `docs/develop/21_code_review_checklist.md`
- `docs/develop/22_risk_control.md`
- `docs/periodPrompt/00_global_rules.md`
- 当前前端、后端、测试、部署相关文件

## 3. 阶段目标

必须完成：

- 前后端完整启动。
- Mock 模式可完整通关。
- 真实 AI 模式可完成 NPC 对话测试。
- 图片生成可测试或有明确 fallback。
- 全中文 UI 检查。
- API Key 不泄露检查。
- 路演演示模式或演示路线。
- 清晰启动命令。
- 最终验收报告。
- 常见失败兜底说明。

## 4. 当前阶段禁止事项

- 禁止为了通过测试删除功能。
- 禁止跳过真实 AI 测试并声称成功。
- 禁止跳过中文 UI 检查。
- 禁止输出 API Key。
- 禁止临时硬编码演示数据破坏正常流程。
- 禁止破坏 Mock 模式。

## 5. 允许修改的文件范围

- 测试文件。
- 启动说明或 README（如项目已有实现需要）。
- 小范围修复影响启动/联调的问题。
- `docs/periodPrompt/reports/stage_10_final_acceptance_report.md`
- `docs/periodPrompt/reports/roadshow_script.md`

## 6. 不允许修改的文件范围

- 不允许大规模重构。
- 不允许修改 PRD。
- 不允许修改 Key 文件。
- 不允许修改 API 合约，除非先强模型审查。

## 7. 具体开发任务

### 任务 1：启动命令确认

- 任务目标：确认前端、后端、测试命令。
- 输出：中文启动说明。
- 验收标准：用户可按命令运行。

### 任务 2：全流程测试

- 任务目标：Mock 和真实 AI 模式均测试。
- 输出：测试记录。
- 验收标准：Mock 可完整通关，真实 AI 至少 NPC 对话成功。

### 任务 3：安全检查

- 任务目标：检查 Key 是否泄露到前端、日志、Markdown。
- 输出：安全检查结论。
- 验收标准：不出现完整 Key。

### 任务 4：路演脚本

- 任务目标：准备 3 分钟中文演示路线。
- 输出：`roadshow_script.md`。
- 验收标准：包含点击路径、讲解点、fallback 方案。

## 8. 数据与接口要求

必须至少验证：

- `GET /api/health`
- `POST /api/session/start`
- `POST /api/investigate`
- `POST /api/dialogue`
- `POST /api/choice`
- `POST /api/ending/resolve`

## 9. 中文化要求

最终 UI 中不得出现英文用户可见文本。必须执行专项扫描，并人工查看主要页面。

## 10. AI 接入要求

必须执行：

- DeepSeek 真实 NPC 对话测试。
- AI JSON 输出测试。
- 监管器测试。
- 图片生成测试或明确 API 不可用时 fallback 记录。

不能用 Mock 冒充真实测试。

## 11. 模型必须执行的自测步骤

- 安装依赖。
- 构建前端。
- 启动后端。
- 启动前端。
- 运行后端测试。
- 运行前端测试或构建。
- Mock 完整通关。
- 真实 AI NPC 测试。
- 监管器违规测试。
- 图片生成测试。
- 中文 UI 检查。
- Key 泄露检查。

## 12. 人工验收方式

1. 按报告中的命令启动项目。
2. 打开前端地址。
3. 按路演脚本点击。
4. 确认线索进入线索栏。
5. 确认 AI 日志显示真实调用成功。
6. 确认监管器可拦截测试违规。
7. 确认图片或占位图可显示。
8. 断开真实 AI 或切换 Mock，确认仍可演示。
