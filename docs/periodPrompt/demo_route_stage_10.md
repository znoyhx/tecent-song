# 《史隙》阶段 10 路演路线

## 1. 演示目标

用三分钟证明《史隙》不是普通问答，而是一条受规则约束的历史悬疑分支叙事：玩家调查红字线索、盘问人物、触发状态推进、做最终抉择，后端按线索与分数判定结局，智能模型只参与人物表达、合规修复或历史回声润色。

## 2. 启动命令

后端：

```powershell
cd backend
python -m uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

前端：

```powershell
cd frontend
npm run dev -- --host 127.0.0.1 --port 5173
```

浏览器打开：`http://127.0.0.1:5173/`。

## 3. 三分钟主路线

- 第一步：在首页选择“明代”，点击生成本局事件，再进入“书坊焚稿案”。
- 第二步：展示开场舞台、人物立绘或剪影、右侧“案卷 / 地点 / 调查”栏。
- 第三步：打开右侧“地点”，切到“书坊后院火场”，点击红字“辽东粮册”，预期获得“烧焦残页”。
- 第四步：继续点击“偏离灯油架的焦痕”或“半枚朱红纸角”，预期右侧线索栏出现至少两条关键线索。
- 第五步：前往“烧毁的刻版间”，向阿沈追问“你昨夜一直在刻坊吗？”，预期底部对话框显示人物回复，并可能获得证言线索。
- 第六步：回到“书坊前厅”调查旧书箱，再去“雨巷”盘问顾闻并出示烧焦残页，预期阶段推进到“关键抉择”。
- 第七步：选择“暗助顾闻护出残页”，进入“暗藏残页”结局页。
- 预期看到：中文结局标题、摘要、结局正文、历史回声、众人去向、重新开始按钮，以及左上“演示证据”面板。

## 4. 三条结局验收路线

| 路线 | 关键操作 | 预期结局 | 预期历史回声 |
| --- | --- | --- | --- |
| 秩序路线 | 推进到关键抉择后选择“将残页交给陆峥” | `ending_order` / 误信告发 | 中文历史回声说明秩序迅速合上裂缝，但真相被压下 |
| 真相路线 | 推进到关键抉择后选择“暗助顾闻护出残页” | `ending_truth` / 暗藏残页 | 中文历史回声引用最终选择、烧焦残页、红印纸角或人物命运 |
| 自保路线 | 推进到关键抉择后选择“焚尽余稿，先保书坊” | `ending_survival` / 焚稿自保 | 中文历史回声说明暂时平安与证据消失之间的代价 |
| 悲剧路线 | 推进到关键抉择后选择“逼阿沈作证换路” | `ending_tragedy` / 身陷诏狱 | 中文历史回声说明风险过高时最弱者先承担代价 |
| 隐藏路线 | 发现封口令、命令矛盾、搜检告示，陆峥与顾问信任达标后选择“顺着封口令反查上游” | `ending_hidden` / 引火反查 | 中文历史回声说明被压下的线头仍有继续追索的可能 |

## 5. 智能模型参与证据

- 人物对话：左上“演示证据”面板显示“人物对话生成”或“本地对话规则”的安全摘要，只显示模块中文名、摘要、成功状态、本地回退、监管通过和时间。
- 历史回声：结局页显示“真实智能模型润色”或“本地中文模板收束”，同时保留中文历史回声正文。
- 监管与回退：演示证据面板显示最近一次监管状态；模型失败、无凭据或输出不合规时走本地中文回退，不阻断主流程。
- 不能声称：不能声称智能模型决定结局编号、阶段跳转或关键线索释放；这些均由后端规则、状态机和线索系统控制。

## 6. 断网 / 无 Key 回退演示

- 若后端未启动：首页会显示中文提示“后端未连通，已切换到离线展示模式”，不会出现空白屏。
- 若真实模型不可用：人物对话与历史回声使用本地中文回退，演示证据面板会记录“本地回退”。
- 若图片缺失：场景图、人物图或线索图使用本地视觉占位或剪影，不影响调查、对话、抉择和结局。
- 路演时只能说明“已触发真实模型 / 已回退 / 已跳过”中的真实结果，不能用本地回退冒充真实模型成功。

## 7. 常见失败与处理

- 后端未启动：先运行后端启动命令，再点击首页的刷新后端按钮。
- 页面仍停在入口：确认已选择“明代”，并点击生成本局事件。
- 找不到关键抉择：按顺序获得烧焦残页、半枚红印纸角、诗稿夹带抄录，或继续盘问顾闻。
- 图片没有显示：这是可接受的视觉回退，继续演示文字、线索和结局。
- 智能模型未成功：查看演示证据面板和结局页来源文案；如显示本地回退，应如实说明。
