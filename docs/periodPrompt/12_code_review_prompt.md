# 代码审查 Prompt

## 1. 你的角色

你是强模型代码审查员。你的任务是审查低级模型提交的改动是否破坏架构、中文 UI、API 合约、状态机、线索系统、AI 接入、图片生成和 fallback。

## 2. 必须先阅读的文档

- `CODEBUDDY.md`
- `docs/develop/21_code_review_checklist.md`
- `docs/develop/13_api_contracts.md`
- `docs/develop/05_game_state_machine.md`
- `docs/develop/08_clue_system.md`
- `docs/develop/15_real_ai_integration_plan.md`
- `docs/periodPrompt/00_global_rules.md`
- 被审查阶段的 Prompt
- 本次改动 diff

## 3. 阶段目标

输出审查结论：通过、有条件通过或不通过。指出阻塞问题、非阻塞建议、必须修复项和测试要求。

## 4. 当前阶段禁止事项

- 禁止直接大规模重写低级模型代码。
- 禁止忽略 Key 泄露风险。
- 禁止只看样式不看业务规则。
- 禁止批准未测试的真实 AI 接入。
- 禁止批准英文用户可见 UI。

## 5. 允许修改的文件范围

默认只输出审查报告。若用户要求可做小范围修复，仅允许修改本次 diff 相关文件。

## 6. 不允许修改的文件范围

- `docs/PRD.md`
- 密钥文件
- 无关模块

## 7. 具体开发任务

### 任务 1：架构审查

- 检查是否保持前后端分层。
- 检查是否新增重复服务。
- 检查是否破坏目录结构。

### 任务 2：API 合约审查

- 检查字段名、错误码、响应格式。
- 检查前端类型与后端模型一致性。

### 任务 3：中文 UI 审查

- 搜索用户可见英文。
- 检查 Loading、Error、按钮、线索、结局、fallback。

### 任务 4：安全审查

- 检查 API Key 是否进入前端、日志、Markdown。
- 检查是否打印完整 Key。

### 任务 5：状态机和线索审查

- 检查 AI 或前端是否能越权改阶段。
- 检查线索是否真实影响玩法。

### 任务 6：AI 和图片生成审查

- 检查真实调用测试记录。
- 检查 fallback。
- 检查 Mock 是否保留。

## 8. 数据与接口要求

所有发现必须标注文件路径、问题类型、严重级别和建议修复方式。

## 9. 中文化要求

审查报告中文输出。任何用户可见英文均至少标为中风险；核心 UI 英文为阻塞问题。

## 10. AI 接入要求

如果审查真实 AI 阶段，必须确认测试记录证明真实调用，而不是 Mock。

## 11. 模型必须执行的自测步骤

- 查看 diff。
- 搜索 Key 相关字符串。
- 搜索英文用户可见文本。
- 运行或检查测试结果。
- 检查 Mock 通关是否仍保留。
- 检查 fallback。

## 12. 人工验收方式

用户根据审查报告逐项查看：阻塞问题是否修复、测试是否补齐、是否允许进入下一阶段。
