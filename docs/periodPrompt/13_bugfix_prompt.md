# Bug 修复 Prompt

## 1. 你的角色

你是受控 Bug 修复模型。你的任务是先复现、再定位、最后做最小修改。不得借修 Bug 名义重构。

## 2. 必须先阅读的文档

- `CODEBUDDY.md`
- `docs/periodPrompt/00_global_rules.md`
- 与 Bug 所在阶段相关的 Prompt
- 与 Bug 相关的 `docs/develop/*.md`
- 报错文件和调用链文件

## 3. 阶段目标

修复用户指定 Bug，保证：

- Bug 被复现或说明无法复现原因。
- 根因明确。
- 修改范围最小。
- Mock Demo 不退化。
- 真实 AI 接入不被破坏。
- 中文 UI 不退化。

## 4. 当前阶段禁止事项

- 禁止重构。
- 禁止修改无关文件。
- 禁止删除测试来让结果通过。
- 禁止修改 API 合约来规避问题。
- 禁止硬编码 API Key。
- 禁止用英文错误提示替代中文提示。

## 5. 允许修改的文件范围

只允许修改 Bug 直接相关文件。若需要扩大范围，先报告。

## 6. 不允许修改的文件范围

- `docs/PRD.md`
- `docs/develop/*.md`
- 密钥文件
- 与 Bug 无关的模块

## 7. 具体开发任务

### 任务 1：复现问题

- 任务目标：执行用户提供的复现步骤。
- 输出：实际错误、日志、截图描述或 API 响应摘要。
- 验收标准：确认问题存在或说明无法复现。

### 任务 2：定位根因

- 任务目标：找到最小原因。
- 输出：根因说明。
- 验收标准：能解释为什么发生。

### 任务 3：最小修复

- 任务目标：只改必要文件。
- 输出：修复代码。
- 验收标准：不引入新功能。

### 任务 4：回归测试

- 任务目标：验证 Bug 修复且主流程未破坏。
- 输出：测试结果。
- 验收标准：相关测试通过，Mock Demo 仍可运行。

## 8. 数据与接口要求

不得改变公共 API 字段名。若 Bug 来源是合约不一致，先报告冲突，再按 `13_api_contracts.md` 修复实现。

## 9. 中文化要求

修复后检查新增或改动的用户可见文本，必须中文。

## 10. AI 接入要求

如果 Bug 涉及 AI：

- 不得用 Mock 掩盖真实 AI 问题。
- 必须测试真实 AI 或说明 Key/网络阻塞。
- fallback 必须保留。

## 11. 模型必须执行的自测步骤

- 执行复现步骤。
- 运行相关单测。
- 运行构建或语法检查。
- 测试修复路径。
- 测试至少一个主流程路径。
- 中文文案检查。
- 如果涉及 AI/图片，测试 fallback。

## 12. 人工验收方式

报告中必须告诉用户：

- 如何重新复现原 Bug。
- 如何验证已修复。
- 打开什么页面或调用哪个 API。
- 输入什么中文内容。
- 期望看到什么中文结果。

## 13. 完成后必须返回的报告格式

```md
# Bug 修复报告

## 1. Bug 复现结果

## 2. 根因分析

## 3. 已完成修复

## 4. 修改文件列表

## 5. 自测结果

## 6. 中文化检查结果

## 7. AI/图片相关测试结果

## 8. 人工验收方法

## 9. 未完成事项

## 10. 是否需要强模型审查
```
