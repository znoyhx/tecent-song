# 21 代码审查清单

## 架构检查

- 是否仍保持前端、后端、数据、文档分层。
- 是否把游戏规则写进前端。
- 是否绕过后端直接调用 AI。
- 是否新增重复服务或平行架构。
- 是否破坏 Mock 与真实 AI 可切换能力。

## 目录检查

- 是否移动或重命名 `docs/PRD.md`、`CODEBUDDY.md`、`docs/develop/`。
- 是否在任务外新增不必要目录。
- 是否把 prompt 硬编码到业务函数而不是模板目录。
- 是否把运行日志写进源码目录。

## API 合约检查

- 请求字段是否符合 `13_api_contracts.md`。
- 响应字段是否稳定。
- 错误响应是否结构化。
- 前端类型是否与后端模型一致。
- 是否出现多个同义字段，例如 `clueId` 和 `clue_id` 混用。

## 状态机检查

- 阶段是否只由后端推进。
- AI 是否能修改阶段或结局。
- intro 是否可能提前释放 reversal 线索。
- choice 是否只能在条件满足后进入。
- ending 后是否还能修改玩法状态。

## AI 输出检查

- Prompt 是否要求 JSON。
- 后端是否解析并校验 JSON。
- 非 JSON 是否有 repair/fallback。
- 是否记录 AI 调用日志。
- 日志是否不含 API Key。

## 线索检查

- 线索是否有唯一 ID。
- 高亮文本是否绑定 clue_id。
- 线索是否有实际 effects 或 unlocks。
- 关键线索是否不会提前释放。
- 线索组合是否检查 required clues。

## NPC 检查

- NPC 是否只知道权限内信息。
- 禁止透露信息是否被监管器使用。
- 信任度变化是否有限制。
- NPC 语气是否符合角色卡。
- 出示线索是否会影响回复。

## 安全检查

- 是否泄露 API Key。
- 是否把 `docs/DeepseekAPIKey` 内容复制进代码。
- 前端是否出现任何模型密钥变量。
- 错误响应是否泄露 prompt、Key、服务器路径。

## Mock Demo 检查

- 无 Key 是否仍可通关。
- Mock 和真实 AI 是否共用相同响应 schema。
- Mock 是否被真实接入删除。
- 路演模式是否可强制 Mock。

## 真实 AI 接入检查

- 是否有超时。
- 是否有最大重试次数。
- 是否有 fallback。
- 是否有 supervisor。
- 是否能关闭真实 AI。

## 路演影响检查

- 主流程是否变慢。
- UI 是否出现空白状态。
- DebugPanel 是否能隐藏。
- 3 分钟演示路线是否仍可走通。
- 监管器演示用例是否仍有效。
