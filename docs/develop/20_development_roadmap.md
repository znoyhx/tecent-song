# 20 开发路线图

## 阶段 0：仓库分析与文档准备

- 阶段目标：明确范围、架构、执行文档。
- 前置条件：已有 PRD。
- 开发任务：生成 `docs/develop/` 文档、确认无现有代码。
- 产出文件：本目录全部文档。
- 验收标准：文档不少于指定 23 份。
- 推荐模型：强模型。
- 风险点：范围膨胀。
- 回滚策略：恢复上一版文档。

## 阶段 1：Mock Demo

- 阶段目标：不依赖真实 AI 完整跑通明代书坊焚稿案。
- 前置条件：API 合约和数据模型确定。
- 开发任务：前后端骨架、Mock 数据、会话、调查、对话、结局。
- 产出文件：`frontend/`、`backend/`、`backend/data/mock/`。
- 验收标准：从开始页到结局可通关。
- 推荐模型：普通模型执行，强模型审查状态机。
- 风险点：Mock 与真实链路分叉。
- 回滚策略：保留 Mock 作为默认模式。

## 阶段 2：前端基础体验

- 阶段目标：形成可展示 UI。
- 前置条件：Mock API 可用。
- 开发任务：GameShell、SceneStage、DialoguePanel、ClueSidebar、EndingView。
- 产出文件：`frontend/src/components/*`。
- 验收标准：线索高亮可点击，NPC 对话可展示。
- 推荐模型：低级/普通模型。
- 风险点：前端自造业务状态。
- 回滚策略：回到简单 UI，保留 API 流程。

## 阶段 3：后端基础服务

- 阶段目标：后端成为状态权威。
- 前置条件：Pydantic 模型完成。
- 开发任务：状态服务、场景服务、线索服务、NPC 服务、结局服务。
- 产出文件：`backend/services/*`、`backend/routers/*`。
- 验收标准：API 单测通过。
- 推荐模型：普通模型，强模型审查。
- 风险点：状态散落在多个服务。
- 回滚策略：回滚到单一 GameStateService。

## 阶段 4：AI NPC 接入

- 阶段目标：真实 AI 可替代 Mock NPC 回复。
- 前置条件：Mock 对话链路稳定。
- 开发任务：AIClient、prompt 模板、DialogueOrchestrator、日志。
- 产出文件：`ai_client.py`、`data/prompts/npc_dialogue.md`。
- 验收标准：有 Key 时真实返回 JSON，无 Key 时 Mock。
- 推荐模型：强模型。
- 风险点：Key 泄露、非 JSON。
- 回滚策略：设置 `USE_MOCK_AI=true`。

## 阶段 5：状态机与线索系统完善

- 阶段目标：线索真实驱动阶段和结局。
- 前置条件：基础服务可用。
- 开发任务：15 条线索、3 组组合、阶段跳转、分数规则。
- 产出文件：`clue_service.py`、`state_machine.py`、clue JSON。
- 验收标准：线索组合推动到 choice。
- 推荐模型：普通模型，强模型审查。
- 风险点：线索提前释放。
- 回滚策略：恢复固定阶段推进规则。

## 阶段 6：一致性监管器

- 阶段目标：拦截 AI 错误。
- 前置条件：AI 对话可用。
- 开发任务：规则检查、RepairAgent、fallback。
- 产出文件：`supervisor.py`、`repair_agent.py`、测试。
- 验收标准：7 类违规全部拦截。
- 推荐模型：强模型。
- 风险点：误杀正常回复。
- 回滚策略：先只启用规则检查，关闭 AI 语义检查。

## 阶段 7：RAG / 知识库接入

- 阶段目标：让 AI 有历史约束上下文。
- 前置条件：关键词知识库可读。
- 开发任务：KeywordRAGService、知识片段、prompt 注入。
- 产出文件：`rag_service.py`、`data/rag_sources/*`。
- 验收标准：“锦衣卫 书坊 文稿”可召回明代规则。
- 推荐模型：普通模型，强模型审查内容。
- 风险点：召回无关内容。
- 回滚策略：关闭 RAG，仅使用固定规则。

## 阶段 8：视觉资产接入

- 阶段目标：提升路演视觉表现。
- 前置条件：UI 主流程稳定。
- 开发任务：占位图、资源解析、视觉提示词、可选生图。
- 产出文件：`assets/*`、`visual_prompt_agent.py`。
- 验收标准：无图片也不崩溃，有图片时正确显示。
- 推荐模型：低级/普通模型。
- 风险点：生图耗时和风格不一致。
- 回滚策略：使用占位剪影。

## 阶段 9：多结局和历史回声

- 阶段目标：完成 5 个结局和 AI 总结。
- 前置条件：choice 阶段稳定。
- 开发任务：EndingRule、HistoryEchoGenerator、结局 UI。
- 产出文件：`ending_service.py`、ending JSON。
- 验收标准：5 结局均可触发。
- 推荐模型：普通模型执行，强模型润色。
- 风险点：结局条件冲突。
- 回滚策略：按 priority 固定判定。

## 阶段 10：测试、部署和路演包装

- 阶段目标：稳定可演示。
- 前置条件：主 Demo 完整。
- 开发任务：测试、演示脚本、日志截图、部署配置。
- 产出文件：测试报告、演示脚本、部署说明。
- 验收标准：3 分钟演示成功，断网可 Mock 演示。
- 推荐模型：强模型。
- 风险点：现场 API 失败。
- 回滚策略：切换 Mock 和本地运行。
