# 22 风险控制

## 工程风险

| 风险 | 表现 | 预防 | 兜底 |
| --- | --- | --- | --- |
| 范围膨胀 | 同时做多朝代、多身份 | 严格执行 MVP 范围 | 只保留明代主线 |
| 目录混乱 | 前后端、数据混在一起 | 使用目录文档 | 强模型审查后重整 |
| API 合约漂移 | 前端字段和后端不一致 | 先写合约和类型 | 冻结合约，逐个修复 |
| 状态重复 | 前端也推进剧情 | 后端为权威 | 删除前端业务推进 |

## AI 输出风险

| 风险 | 表现 | 预防 | 兜底 |
| --- | --- | --- | --- |
| 非 JSON | 模型输出散文 | JSON-only prompt + schema | RepairAgent / fallback |
| 剧透 | NPC 直接说出真相 | forbidden_disclosure | 监管器拦截 |
| 人设漂移 | 胆小 NPC 突然全盘交代 | 角色卡 + 信任度 | 修复或 Mock |
| 错朝代 | 出现现代/清代元素 | 禁词 + RAG | fallback |
| 延迟高 | 等待太久 | 超时和 loading | 切 Mock |

## RAG 风险

| 风险 | 表现 | 预防 | 兜底 |
| --- | --- | --- | --- |
| 召回无关 | prompt 混入其他朝代 | 先按 dynasty 过滤 | 关闭 RAG |
| 内容不可信 | AI 用错资料 | 来源等级 | 只用规则库 |
| 接入过重 | 向量库拖慢开发 | 先关键词检索 | 后续替换接口 |

## 成本风险

| 风险 | 表现 | 预防 | 兜底 |
| --- | --- | --- | --- |
| AI 调用太多 | 开发成本上升 | Mock 默认开启 | 批量缓存回复 |
| 生图成本高 | 多次生成不满意 | 先做提示词和占位 | 手工精选少量图 |
| 强模型滥用 | 简单任务也用强模型 | 任务分级 | 强模型只审查关键点 |

## 时间风险

| 风险 | 表现 | 预防 | 兜底 |
| --- | --- | --- | --- |
| 先做底层过久 | 没有可演示页面 | 阶段 1 先 Mock 通关 | 砍 RAG 和生图 |
| 调 AI 调太久 | 卡在模型格式 | 先固定 Mock schema | 用 fallback 路演 |
| UI 打磨过度 | 玩法没做完 | P0 优先线索和状态 | 使用占位 UI |

## 美术风险

| 风险 | 表现 | 预防 | 兜底 |
| --- | --- | --- | --- |
| 风格不一致 | NPC/背景像不同游戏 | 统一提示词 | 剪影化处理 |
| 图片缺失 | 页面空白 | AssetResolver | CSS 占位 |
| 生图 URL 过期 | 图片丢失 | 立即下载保存 | 本地占位图 |

## 展示风险

| 风险 | 表现 | 预防 | 兜底 |
| --- | --- | --- | --- |
| 现场网络失败 | AI 无响应 | 预置 Mock 模式 | 本地演示 |
| 评委看不懂 AI 价值 | 只像文字游戏 | 展示日志和监管器 | 准备讲解脚本 |
| 演示路径太长 | 3 分钟走不完 | 固定演示路线 | 准备存档状态 |

## 低级模型开发风险

| 风险 | 表现 | 预防 | 兜底 |
| --- | --- | --- | --- |
| 乱改架构 | 重构目录、删服务 | Prompt 限定文件 | git diff 审查 |
| 改错合约 | 字段名漂移 | 合约文档固定 | 强模型审查 |
| 删除 Mock | 真实 AI 不稳 | 明确禁止 | 恢复 Mock 分支 |
| 写死密钥 | Key 泄露 | 禁止读取 Key 文件 | 立即移除并轮换 |

## 最高优先级风险清单

1. 主 Demo 跑不通。
2. AI 输出破坏状态机。
3. 线索只是装饰，没有影响玩法。
4. API Key 泄露到前端或日志。
5. 低级模型重构导致文档和实现脱节。

## 总体兜底策略

- 永远保留 Mock 模式。
- 永远以后端状态机为权威。
- 真实 AI 失败不阻塞通关。
- RAG 和生图都可关闭。
- 每个阶段交付后立刻做一次完整通关测试。
